什么是RMI

远程调用，为什么会出现这个东西，其实就是客户端想要运行别的地方的java代码，这是一个刚需。

这里有个图，大致原理就是RMI有一个服务端，我们的客户客户端想要运行一个没有的代码，去向RMI服务端要，比如一个对象，这个怎么去要呢？他怎么知道要哪个呢？或者说，服务端怎么知道要给哪个，这里就出现了一个叫注册中心的东西，所有的服务要通过注册中心进行查询，注册中心也会为这些对象动态的绑定端口，这样就不会出现端口冲突的情况
